Food Delivery System (Startup, Low Budget):

Components:
- NGINX Load Balancer
- Kong API Gateway
- JWT Auth Service
- Order Service (FastAPI)
- Delivery Service (FastAPI)
- Restaurant Service (FastAPI)
- RabbitMQ Queue
- Redis Cache
- PostgreSQL Primary
- PostgreSQL Read Replica
- Amazon S3
- Prometheus + Grafana

Architecture:
Client requests flow through NGINX and Kong API Gateway which handles routing and JWT authentication. Order Service manages order lifecycle and publishes events to RabbitMQ for asynchronous delivery assignment. Delivery Service consumes queue events and updates delivery status. Redis caches restaurant menus and delivery zones to reduce database load.

Scaling:
System scales horizontally by adding EC2 instances behind the load balancer. PostgreSQL uses a read replica for read-heavy queries. Redis handles hot data caching to reduce database pressure. RabbitMQ enables asynchronous processing to smooth traffic spikes.

Constraints:
- Works under $2000/month
- Avoid Kubernetes and Kafka
- Suitable for 50–100 RPS
