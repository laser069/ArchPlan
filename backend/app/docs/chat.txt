Real-time Chat System:

Components:
- NGINX
- WebSocket Gateway
- Auth Service
- Chat Service
- RabbitMQ Queue
- Redis Cache
- PostgreSQL DB

Architecture:
WebSocket gateway maintains persistent connections. Messages are pushed through RabbitMQ and stored in PostgreSQL. Redis caches recent chats.

Scaling:
WebSocket servers scale horizontally. Redis handles real-time message caching. Queue decouples message processing.

Constraints:
- Low latency
- Real-time delivery
